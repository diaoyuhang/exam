# 2026 Computer Industry Examination Prediction Paper

## 一、考试说明

- 试卷名称：2026 年计算机行业考试预测卷（第五版）
- 考试时间：120 分钟
- 试卷总分：60 分
- 注意事项：
  1. 本试卷分为单项选择题、多项选择题、简答题、实务题四个部分。
  2. 单项选择题只有 1 个正确答案，多项选择题有 2 个或以上正确答案，错选不得分，漏选酌情给分。
  3. 简答题与实务题应结合题意作答，条理清晰，术语准确。
  4. 本预测卷在延续历年命题风格的基础上，结合近年网络安全、云边协同、数据治理与软件测试等方向命题。

## 五、实务题（共 3 题，共 40 分）

### 1. 校园网络升级改造设计题（12 分）

某职业院校拟对现有校园网络进行升级改造。学校现有教学楼、实训楼、图书馆、宿舍区和办公区，原网络存在带宽不足、无线覆盖不均、访问控制粗放、网络故障定位困难等问题。学校要求新网络能够支持在线教学、实验平台访问、视频监控、宿舍无线接入和办公业务，并具备较好的安全隔离能力、统一认证能力和运行监控能力。

请完成下列任务：

1. 分析该校园网络涉及的主要网络区域、主要设备类型及其相互关系。
2. 结合题意，写出网络升级中至少 4 项关键技术或建设措施。
3. 说明 VLAN 划分、统一认证和网络监控在该项目中的主要作用。

### 2. 医疗预约与排班系统设计题（14 分）

某医院拟建设“医疗预约与排班系统”。患者可在线查询科室、医生、可预约时段并提交预约申请；系统需记录患者基本信息、医生信息、科室信息、排班计划、预约记录和就诊状态；医院管理部门还需统计医生接诊量、预约率、爽约率等数据，用于调整排班与资源配置。

请完成下列任务：

1. 分析该系统的核心实体，并给出主要实体之间的关系。
2. 设计该系统的关系模式，至少包含患者、科室、医生、排班计划、预约记录等核心数据对象。
3. 说明在该系统中，如何通过数据库设计减少预约冲突和数据不一致问题。

### 3. 应用日志分析与程序实现题（14 分）

某单位拟开发一段应用日志分析程序，对接口调用结果进行初步判断。规则如下：若某接口在 5 分钟内调用失败次数超过阈值，则进一步检查其平均响应时间是否同时超过设定值；若两个条件都满足，则判定为严重异常；若仅失败次数超阈值，则判定为一般异常；否则视为正常。

现给出伪代码如下：

```text
function checkApiStatus(failCount, avgTime, failLimit, timeLimit)
begin
    if failCount > failLimit then
        if avgTime > timeLimit then
            return "critical"
        else
            return "warning"
        end if
    else
        return "normal"
    end if
end
```

请完成下列任务：

1. 简述该程序的功能，并说明其主要体现了哪一种基本程序结构。
2. 分析该程序作为异常判断规则可能存在的局限性。
3. 从系统运维与监控角度，提出至少 3 项可进一步提高异常识别效果的改进措施。
4. 若系统还需记录每次接口检测的“接口名称、统计时间段、失败次数、平均响应时间、判定结果、处置状态”，请设计一张数据表，并写出主要字段。
