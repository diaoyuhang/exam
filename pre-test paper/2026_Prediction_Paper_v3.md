# 2026 Computer Industry Examination Prediction Paper

## 一、考试说明

- 试卷名称：2026 年计算机行业考试预测卷（第三版）
- 考试时间：120 分钟
- 试卷总分：60 分
- 注意事项：
  1. 本试卷分为单项选择题、多项选择题、简答题、实务题四个部分。
  2. 单项选择题只有 1 个正确答案，多项选择题有 2 个或以上正确答案，错选不得分，漏选酌情给分。
  3. 简答题与实务题应结合题意作答，条理清晰，术语准确。
  4. 本预测卷在延续历年考试风格的基础上，重点结合近年数据工程、云原生运维、网络安全治理和物联网应用等方向命题。

## 五、实务题（共 3 题，共 40 分）

### 1. 智慧停车管理系统设计题（12 分）

某市拟建设“智慧停车管理系统”。系统通过地磁传感器和摄像设备采集停车位状态，车主可通过小程序查询空闲车位、预约停车、在线支付；管理部门需要统计各区域车位利用率、异常占位情况和高峰时段数据；系统还需记录停车订单、设备状态、收费规则和告警信息。

请完成下列任务：

1. 分析该系统的核心实体，并给出主要实体之间的关系。
2. 设计该系统的关系模式，至少包含停车场、车位、车辆、停车订单、采集设备、告警记录等核心数据对象。
3. 说明该系统中“实时状态采集”和“历史数据统计分析”在技术处理方式上的主要区别。

### 2. 数据仓库与报表平台建设题（14 分）

某制造企业现有采购、库存、生产、销售等多个业务系统，数据分散且口径不一致，管理层难以及时获得统一经营报表。企业计划建设数据仓库和报表分析平台，实现跨系统数据整合、统一指标口径、按日生成主题报表，并支持按地区、产品、月份等维度进行分析。

请完成下列任务：

1. 说明该企业建设数据仓库而不是直接在业务库上做复杂统计分析的主要原因。
2. 结合题意，写出数据仓库建设过程中至少 4 个关键环节或关键技术。
3. 说明主题建模、数据清洗和统一指标口径分别解决什么问题。

### 3. 网络访问控制与程序分析题（14 分）

某单位拟开发一段访问控制程序，对进入内网系统的请求进行初步检查。规则如下：若请求来源 IP 属于允许网段，则继续检查访问时间是否在办公时段内；若在办公时段内，再检查用户角色是否具备访问目标资源的权限；只有全部满足条件时，才允许访问。

现给出伪代码如下：

```text
function allowAccess(ip, time, role, resource)
begin
    if inWhiteList(ip) then
        if inWorkTime(time) then
            if hasPermission(role, resource) then
                return true
            else
                return false
            end if
        else
            return false
        end if
    else
        return false
    end if
end
```

请完成下列任务：

1. 简述该程序的功能，并说明其主要体现了哪一种基本程序结构。
2. 分析该程序在访问控制方面仍可能存在的不足。
3. 从网络安全角度，提出至少 3 项可进一步增强访问控制效果的措施。
4. 若系统还需记录每次访问申请的“来源 IP、访问时间、用户角色、目标资源、是否放行、拒绝原因”，请设计一张数据表，并写出主要字段。

## 六、命题特点说明

本预测卷相对前两版刻意避开了“知识库问答平台”“企业微服务上云”“登录或文件上传安全程序”等已出现过的题材，改用新的业务场景进行预测，主要特点如下：

- 保持近年稳定结构：10 道单选题、5 道多选题、2 道简答题、3 道实务题，总分 60 分。
- 基础题覆盖网络、数据库、软件工程、容器、消息队列、访问控制等常见核心知识。
- 简答题转向数据仓库与 DevOps，贴合近年企业数字化建设中的高频实践。
- 实务题更换为“智慧停车”“数据仓库报表平台”“网络访问控制程序分析”，与前两版题材区分明显。
- 难度上仍保持基础题为主、综合题提升、实务题拉开层次的命题节奏。
